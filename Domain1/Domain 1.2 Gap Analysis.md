# 🔐 1.2 Gap Analysis 

1️⃣ What is a Gap Analysis? 🕳️

Definition:
A gap analysis compares:

Current state (where you are) 📍
Desired state (where you want to be) 🎯

👉 Goal: Identify what’s missing (“the gap”) in your security posture and how to fix it 🔧

2️⃣ Why Gap Analysis Matters in Security 🛡️
Organizations constantly evolve → security must keep up 🔄
Helps identify:
Weaknesses in systems ⚠️
Missing controls ❌
Policy failures 📉
Provides a roadmap for improvement 🗺️

Reality:

Not a quick task ⏳
Can take weeks, months, or even years 📆
Involves:
Multiple teams 👥
Data collection 📊
Project planning 📋

3️⃣ The Importance of a Baseline 📏

Before performing a gap analysis, you need a baseline.

Baseline = Target Standard 🎯

A reference point for what “secure” looks like 🔐

Common Baselines:
NIST SP 800-171 🇺🇸
Focus: Protecting Controlled Unclassified Information (CUI) 🔒
ISO/IEC 27001 🌐
Focus: Information Security Management Systems (ISMS) 🏢
Custom baseline ⚙️
Built for your organization’s needs

4️⃣ What Gets Evaluated? 🔍
👥 People
Skills and experience in IT/security 🧠
Training and certifications 🎓
Knowledge of policies and procedures 📘
⚙️ Processes
Security policies 📜
Procedures and workflows 🔄
Compliance with standards ✅
🖥️ Systems
Current infrastructure 💻
Configurations ⚙️
Security controls in place 🔐

5️⃣ The Analysis Process 🔄
Step 1: Compare Current State vs Baseline 📊
Identify gaps (weaknesses) 🕳️
Example:
Baseline requires MFA → company only uses passwords → gap identified ❌
Step 2: Break Down Security into Smaller Pieces 🧩

Instead of analyzing everything at once:

Start broad → then go detailed 🔍

Example:

Access Control (broad) 🔑
User registration 👤
User de-registration ❌
Privileged access management 🔐
Access reviews 📋

👉 This makes analysis more manageable and precise 🎯

6️⃣ Creating the Gap Analysis Report 📄

After gathering all data, you create a final report.

The report includes:
Current state 📍
Desired state (baseline) 🎯
Identified gaps 🕳️
Recommendations 💡
Action plan (how to fix gaps) 🔧

7️⃣ The “Path Forward” (Most Important Part) 🚀

Knowing the gap is easy.
Fixing it is the challenge.

Requires:
⏳ Time
💰 Budget
🛠️ New tools or systems
🔄 Change management (implementing safely)

8️⃣ Visual Prioritization (Red / Yellow / Green) 🚦

Gap analysis often uses a color-coded system:

🟢 Green → Meets baseline (good)
🟡 Yellow → Partially meets baseline (needs improvement)
🔴 Red → Does NOT meet baseline (high priority)

How to prioritize:
Fix Red first (highest risk) 🔥
Then Yellow ⚠️
Maintain Green ✅

9️⃣ Example Scenario 🏢

Organization has 7 locations:

Location	Security Status
Site A	🟢
Site B	🔴
Site C	🟡

👉 Best strategy:

Start fixing Site B (Red) → biggest impact 🔥
Then improve Site C (Yellow) ⚠️
Maintain Site A (Green) ✅

🔟 Key Takeaways 📌
Gap analysis = Current vs Desired 🔄
Requires a baseline (NIST, ISO, or custom) 📏
Evaluates:
People 👥
Processes ⚙️
Technology 💻
Produces a detailed report + improvement plan 📄
Uses prioritization (Red/Yellow/Green) 🚦
Focus is not just identifying gaps—but closing them 🔧

🧠 Memory Anchors
🕳️ “Gap” = hole in your security
🗺️ Gap Analysis = GPS
Current location → Destination → Route to get there
🚦 Red = Fix NOW, Yellow = Improve, Green = Good

🧾 Flashcards (Active Recall) 🎴
Q: What is a gap analysis?
A: A comparison between current security state and desired baseline.
Q: Why is a baseline important?
A: It defines the target security standard to measure against.
Q: Name two common security baselines.
A: NIST SP 800-171 and ISO/IEC 27001.
Q: What are the three main areas evaluated?
A: People, Processes, Systems.
Q: What does a red status indicate?
A: High risk, does not meet baseline.
Q: What is the hardest part of gap analysis?
A: Implementing the improvements (time, cost, changes).

A: To analyze and manage complexity more effectively.

✅ Final Insight:
Gap analysis is not just about finding problems—it’s about building a realistic, structured path to stronger security. 🚀
