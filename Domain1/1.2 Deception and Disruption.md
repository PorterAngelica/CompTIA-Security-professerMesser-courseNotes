# 🕵️‍♂️ 1.2: Deception & Disruption

---

## 1️⃣ What is Deception in Security?

**Definition:**
Using **fake systems, data, or environments** to:

* Trick attackers
* Observe their behavior
* Detect intrusions early

👉 Instead of only blocking attackers, you **mislead and study them**

---

## 2️⃣ Honeypots

### 🍯 Definition:

A **honeypot** is a decoy system designed to:

* Attract attackers
* Keep them engaged
* Monitor their actions

### 🎯 Purpose:

* Detect attacks
* Study attacker techniques
* Distract attackers from real systems

### ⚙️ Key Characteristics:

* Not part of production systems
* Looks like a real system
* Often targeted by automated attacks (bots)

---

### 🧠 Example:

* Fake server with “vulnerabilities”
* Attacker spends time hacking it
* Meanwhile, real systems stay protected

---

## 3️⃣ The “Arms Race”

* Attackers try to **detect honeypots**
* Defenders make honeypots more **realistic**

👉 Constant evolution on both sides

---

## 4️⃣ Honeynets

### 🌐 Definition:

A **honeynet** = multiple honeypots combined into a **full fake network**

### 🏗️ Includes:

* Servers
* Workstations
* Routers
* Firewalls

### 🎯 Purpose:

* Create a **realistic environment**
* Keep attackers engaged longer
* Gather more detailed intelligence

---

### 🧠 Think of it like:

* Honeypot = fake house
* Honeynet = fake neighborhood

---

## 5️⃣ Honeyfiles

### 📄 Definition:

Fake files designed to look **important or sensitive**

### Examples:

* `passwords.txt`
* `confidential_data.xlsx`

### 🎯 Purpose:

* Attract attackers
* Trigger alerts if accessed

👉 Important rule:

* **No legitimate user should access these files**
* If accessed → **likely malicious activity**

---

## 6️⃣ Honeytokens

### 🧬 Definition:

**Fake data that is trackable**

### Examples:

* Fake API keys
* Fake email addresses
* Fake database records

### 🎯 Purpose:

* Track data leaks
* Identify attackers
* Trace where stolen data appears

---

### 🧠 Example:

* Fake email placed in a file
* Later appears online → you know it was stolen

---

## 7️⃣ Detection & Alerting

* Honeypots and honeyfiles are monitored
* If accessed → alerts are triggered

👉 This provides:

* Early warning
* High-confidence detection (low false positives)

---

## 8️⃣ Types of Deception Tools

| Tool       | Description          | Purpose                    |
| ---------- | -------------------- | -------------------------- |
| Honeypot   | Single fake system   | Attract attackers          |
| Honeynet   | Network of honeypots | Realistic environment      |
| Honeyfile  | Fake sensitive file  | Detect unauthorized access |
| Honeytoken | Trackable fake data  | Detect data leaks          |

---

## 9️⃣ Advantages & Limitations

### ✅ Advantages:

* Detect unknown attacks
* Low false positives
* Gain attacker intelligence
* Distract attackers

### ⚠️ Limitations:

* Must be isolated (or attacker could pivot)
* Requires maintenance
* Skilled attackers may detect them

---

## 🔟 Key Takeaways

* Deception = **proactive defense strategy**
* Honeypots attract attackers
* Honeynets increase realism
* Honeyfiles detect unauthorized access
* Honeytokens track data leaks
* These tools help you **learn from attackers**

---

## 🧾 Flashcards (Active Recall)

1. **Q:** What is a honeypot?
   **A:** A fake system designed to attract and monitor attackers.

2. **Q:** What is a honeynet?
   **A:** A network of honeypots creating a realistic fake environment.

3. **Q:** What is a honeyfile?
   **A:** A fake file used to detect unauthorized access.

4. **Q:** What is a honeytoken?
   **A:** Trackable fake data used to identify leaks or attackers.

5. **Q:** Why are honeypots useful?
   **A:** They detect attacks and provide insight into attacker behavior.

6. **Q:** What happens if a honeyfile is accessed?
   **A:** It likely indicates malicious activity and triggers alerts.

Traditional security says: **“Keep attackers out.”**
