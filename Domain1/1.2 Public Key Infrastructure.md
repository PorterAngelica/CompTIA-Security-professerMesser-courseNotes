# 🔐 1.2 : Public key infrastructure

---

## 1️⃣ What is PKI (Public Key Infrastructure)?

**Definition:**
A system of:

* Policies
* Procedures
* Hardware & software

Used to:

* Create
* Manage
* Distribute
* Store
* Revoke digital certificates 

---

### 🎯 Purpose of PKI:

* Verify identity (users/devices)
* Enable secure communication
* Support digital certificates

---

## 2️⃣ Symmetric Encryption

### 🔑 Definition:

Uses **ONE key** for:

* Encryption
* Decryption

👉 Same key = shared secret

---

### 📌 Key Terms:

* Secret key
* Shared key

---

### ⚙️ Characteristics:

* Very **fast**
* Low overhead
* Requires **secure key sharing**

---

### ⚠️ Problem:

* **Scalability issue**
* Hard to manage keys for many users

👉 Example:
10 users → many keys needed

---

### 🧠 Memory Anchor:

* 🔑 “One key opens and locks”

---

## 3️⃣ Asymmetric Encryption (Public Key Cryptography)

### 🔐 Definition:

Uses **TWO keys**:

* Public key
* Private key

---

### 🔑 Key Roles:

| Key         | Purpose                        |
| ----------- | ------------------------------ |
| Public Key  | Encrypt (shared with everyone) |
| Private Key | Decrypt (kept secret)          |

---

### ⚙️ Key Concepts:

* Keys are **mathematically related**
* Generated **together**
* Cannot derive private key from public key

---

### 📩 How It Works:

1. Bob wants to send message to Alice
2. Bob uses **Alice’s public key** to encrypt
3. Sends ciphertext
4. Alice uses **private key** to decrypt

👉 Only Alice can read it

---

### 🧠 Memory Anchor:

* 📬 “Public key = mailbox, Private key = key to open mailbox”

---

## 4️⃣ Symmetric vs Asymmetric (Comparison)

| Feature  | Symmetric        | Asymmetric             |
| -------- | ---------------- | ---------------------- |
| Keys     | 1 key            | 2 keys                 |
| Speed    | Fast             | Slower                 |
| Security | Key sharing risk | More secure            |
| Use Case | Bulk encryption  | Key exchange, identity |

---

### 🔥 IMPORTANT:

👉 Real systems use **both together**

* Asymmetric → secure key exchange
* Symmetric → fast data encryption

---

## 5️⃣ Key Generation Process

* Public & private keys created **at the same time**
* Uses:

  * Large prime numbers
  * Randomization

👉 Done once per user/device

---

## 6️⃣ Key Management (Enterprise Level)

### Problem:

* Thousands of users → thousands of keys

---

### Solution: Key Management Systems

#### 🔐 Key Escrow

* Store private keys securely
* Allows access if:

  * User leaves company
  * Data recovery needed

---

### ⚠️ Tradeoff:

* Convenience vs security
* Sharing private keys introduces risk

---

## 7️⃣ Real-World Example

* Alice creates key pair
* Shares public key publicly
* Bob encrypts message
* Only Alice decrypts it

👉 Even if intercepted → unreadable

---

## 8️⃣ Key Takeaways

* PKI manages **certificates and trust**
* Symmetric:

  * Fast
  * Not scalable
* Asymmetric:

  * Secure
  * Slower
* Both are used together
* Private key must ALWAYS be protected

---

## 🧠 Memory Anchors

* 🔑 Symmetric = “one key for everything”
* 📬 Asymmetric = “mailbox system”
* 🏢 PKI = “certificate management system”
* 🔐 Private key = “never share”

---

## 🧾 Flashcards (Active Recall)

1. **Q:** What is PKI?
   **A:** A system that manages digital certificates and encryption keys.

2. **Q:** What is symmetric encryption?
   **A:** Encryption using the same key for encrypting and decrypting.

3. **Q:** What is asymmetric encryption?
   **A:** Encryption using a public/private key pair.

4. **Q:** Which key is shared publicly?
   **A:** Public key.

5. **Q:** Which key must remain secret?
   **A:** Private key.

6. **Q:** Why is symmetric encryption still used?
   **A:** It is faster and more efficient.

7. **Q:** What is key escrow?
   **A:** Storing private keys for recovery or access.

---

## ✅ Final Insight

👉 **Encryption = balance of speed and security**

* Symmetric = speed
* Asymmetric = trust

Together → secure communication at scale
