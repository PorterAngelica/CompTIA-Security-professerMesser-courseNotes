# 🔐 1.4 : Data Encryption 

---

## 1️⃣ Data Encryption Types

### 💾 Data at Rest

**Definition:**
Data stored on:

* Hard drives
* SSDs
* Databases

👉 Must be encrypted to protect against physical theft

---

### 🔒 Full Disk Encryption (FDE)

* Encrypts **entire drive/volume**
* Examples:

  * Windows: BitLocker
  * macOS: FileVault

---

### 📄 File-Level Encryption

* Encrypts **individual files only**
* Example:

  * Windows: EFS (Encrypting File System)

---

## 2️⃣ Database Encryption

### 🗄️ Transparent Database Encryption

* Encrypts entire database using **symmetric key**
* Data decrypted when accessed

---

### 📊 Column-Level Encryption

* Encrypts **specific fields only** (e.g., SSN)

### ✅ Benefit:

* Faster searches (no need to decrypt everything)

---

### 🧠 Example:

* Names = plaintext
* SSN = encrypted

---

## 3️⃣ Data in Transit

### 🌐 HTTPS

* Encrypts browser traffic
* Uses SSL/TLS

---

### 🔐 VPN (Virtual Private Network)

* Creates **encrypted tunnel**

#### Types:

* Remote access VPN → user to network
* Site-to-site VPN → network to network

#### Protocols:

* SSL/TLS (client VPN)
* IPsec (site-to-site VPN)

---

## 4️⃣ Encryption Algorithms

### ⚠️ IMPORTANT RULE:

👉 Both sides must use the **same algorithm**

---

### Examples:

* DES (older, weaker)
* AES (modern, secure standard)

👉 You CANNOT:

* Encrypt with DES → Decrypt with AES

---

## 5️⃣ Algorithm vs Key (CRITICAL CONCEPT)

### 🔓 Algorithm:

* Publicly known
* Trusted because it’s tested

### 🔑 Key:

* Secret
* Determines security

---

### 🧠 Analogy:

* Lock = algorithm
* Key = actual access

👉 Knowing the lock design ≠ opening the lock

---

## 6️⃣ Key Security

### 🔐 Rule:

👉 **Protect the key at all costs**

If attacker gets the key:

* Encryption is useless

---

## 7️⃣ Brute Force Attacks

### ⚔️ Definition:

Attacker tries **every possible key combination**

---

### 🛡️ Defense:

* Use **longer keys**

### Examples:

* Symmetric: 128-bit or higher
* Asymmetric: 3072-bit or higher

---

## 8️⃣ Key Stretching (Key Strengthening)

### 🔁 Definition:

Applying encryption multiple times

### Example:

* Hash(password)
* Hash(hash(password))
* Repeat many times

---

### 🎯 Purpose:

* Slow down brute force attacks
* Increase attack complexity

---

## 9️⃣ Key Takeaways

* Data must be protected:

  * At rest
  * In transit
* Full disk encryption protects entire system
* File/column encryption protects specific data
* Same algorithm required on both sides
* Security depends on **key secrecy**, not algorithm secrecy
* Longer keys = stronger protection
* Key stretching = extra security layer

---

## 🧠 Memory Anchors

* 💾 **At rest = stored data**
* 🌐 **In transit = moving data**
* 🔐 **VPN = encrypted tunnel**
* 🔑 **Key = real security**
* 🔁 **Key stretching = slow attackers down**

## 🧾 Flashcards (Active Recall)

1. **Q:** What is data at rest?
   **A:** Data stored on a device (disk, SSD, database).

2. **Q:** What is full disk encryption?
   **A:** Encrypting an entire storage device.

3. **Q:** What is EFS?
   **A:** File-level encryption in Windows.

4. **Q:** What does HTTPS protect?
   **A:** Data in transit over the web.

5. **Q:** What is a VPN?
   **A:** An encrypted tunnel between devices/networks.

6. **Q:** Why must both sides use the same algorithm?
   **A:** To properly encrypt and decrypt data.

7. **Q:** What is more important: algorithm or key?
   **A:** The key.

8. **Q:** What is key stretching?
   **A:** Repeated encryption to slow brute force attacks.

---

## ✅ Final Insight

👉 Encryption is NOT about hiding the method
👉 It’s about protecting the **key**

If the key is safe → your data is safe
If the key is exposed → everything is compromised
