# 🔐 1.4 : Encryption Technologies

---

## 1️⃣ Trusted Platform Module (TPM)

### 💻 Definition:

A **hardware chip on a device** that performs cryptographic operations.

👉 Built into:

* Motherboards
* Laptops
* Desktops

---

### 🔑 Functions:

* Generate encryption keys
* Store keys securely
* Provide random number generation
* Support full disk encryption (e.g., BitLocker)

---

### 🔒 Security Features:

* Keys are **device-specific**
* Stored in **secure hardware memory**
* Protected from brute force attacks 

---

### 🧠 Use Case:

* Laptop encryption
* Secure boot
* Device identity

---

## 2️⃣ Hardware Security Module (HSM)

### 🏢 Definition:

A **centralized hardware system** used for large-scale cryptography.

---

### 📦 Used for:

* Data centers
* Enterprise environments
* Cloud systems

---

### 🔑 Functions:

* Store large numbers of keys
* Perform encryption/decryption
* Handle certificates

---

### ⚙️ Features:

* High performance (hardware acceleration)
* Redundant (power, network)
* Centralized key storage

---

### 🧠 Analogy:

* TPM = safe in your house
* HSM = bank vault for entire company

---

## 3️⃣ TPM vs HSM (Comparison)

| Feature  | TPM             | HSM                             |
| -------- | --------------- | ------------------------------- |
| Scope    | Single device   | Enterprise-wide                 |
| Location | Local hardware  | Centralized system              |
| Use Case | Disk encryption | Key management for many systems |

---

## 4️⃣ Key Management Systems (KMS)

### 🧠 Definition:

Central system to manage all encryption keys

---

### 🎯 Functions:

* Create keys
* Store keys
* Assign keys to users/devices
* Rotate keys automatically
* Log usage

---

### 🔄 Key Rotation:

* Regularly replace keys
* Reduces risk if key is compromised

---

### 📊 Benefits:

* Central control
* Visibility (reports/logs)
* Separation of keys from data

---

## 5️⃣ Types of Keys Managed

* SSL/TLS keys (web servers)
* SSH keys (remote access)
* BitLocker keys (disk encryption)
* Active Directory keys

---

## 6️⃣ Secure Enclave

### 🧩 Definition:

A **dedicated security processor inside a device**

👉 Separate from main CPU

---

### 🔒 Purpose:

* Protect sensitive data
* Perform secure cryptographic operations

---

### ⚙️ Features:

* Own processor
* Own memory
* Secure boot process
* True random number generator
* Hardware encryption (AES)
* Built-in root keys

---

### 📱 Found in:

* Smartphones
* Laptops
* Modern computers

---

### 🧠 Key Idea:

Even if attacker has your device → data stays protected

---

## 7️⃣ Why These Technologies Matter

### Problem:

* Data is everywhere (phones, laptops, cloud)
* Attackers constantly evolve

---

### Solution:

* Hardware-based security
* Centralized key management
* Secure processing environments

---

## 8️⃣ Key Takeaways

* TPM = **device-level security chip**
* HSM = **enterprise-level key storage system**
* KMS = **manages all keys centrally**
* Secure enclave = **isolated processor for protecting data**

---

## 🧠 Memory Anchors

* 💻 **TPM = device safe**
* 🏢 **HSM = company vault**
* 🔑 **KMS = key manager**
* 🧠 **Secure enclave = brain protecting secrets**

## 🧾 Flashcards (Active Recall)

1. **Q:** What is a TPM?
   **A:** A hardware chip that performs cryptographic operations on a device.

2. **Q:** What is an HSM?
   **A:** A centralized hardware system for managing encryption keys.

3. **Q:** What is key rotation?
   **A:** Regularly changing encryption keys to improve security.

4. **Q:** What is a secure enclave?
   **A:** A dedicated processor that protects sensitive data.

5. **Q:** What is the difference between TPM and HSM?
   **A:** TPM is for one device; HSM is for enterprise environments.

6. **Q:** Why use a key management system?
   **A:** To centrally manage, store, and monitor keys.

## ✅ Final Insight

👉 **Modern security = hardware + software working together**

* Software alone is not enough
* Hardware security (TPM, HSM, Secure Enclave) = **stronger protection**
