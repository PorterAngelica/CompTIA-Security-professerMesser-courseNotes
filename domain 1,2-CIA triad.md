## Domain 1.2 – The CIA Triad

---

## 1. Course Notes (Simple / Messer Style)

### What is the CIA Triad

The CIA Triad is the foundation of cybersecurity.

It defines the three main goals of security:

- Confidentiality  
- Integrity  
- Availability  

### Simple way to remember it
> CIA = what we are trying to protect

---

## Confidentiality

Confidentiality means keeping data private and preventing unauthorized access.

Only authorized users should see the data.

### Simple way to remember it
> Confidentiality = keep data secret

### Examples
- Encryption  
- Passwords  
- Access control  
- File permissions  

### Example
A company encrypts customer data so attackers cannot read it.  
👉 This protects **confidentiality**

---

## Integrity

Integrity means ensuring data is accurate and has not been altered.

Data should not be modified without authorization.

### Simple way to remember it
> Integrity = data is correct and unchanged

### Examples
- Hashing  
- Digital signatures  
- File integrity monitoring  

### Example
A system detects that a file was modified unexpectedly.  
👉 This protects **integrity**

---

## Availability

Availability means systems and data are accessible when needed.

Users should be able to access resources without interruption.

### Simple way to remember it
> Availability = systems are up and working

### Examples
- Backups  
- Redundancy  
- Load balancing  
- Failover systems  

### Example
A website stays online even during high traffic.  
👉 This protects **availability**

## 2️⃣ AAA (VERY IMPORTANT 🔥)

### What is AAA

AAA stands for:

- Authentication  
- Authorization  
- Accounting  

### Simple way to remember it
> AAA = Who you are, what you can do, and what you did

---

### Authentication

Authentication verifies your identity.

#### Simple way to remember it
> Authentication = prove who you are

#### Examples:
- Passwords  
- Biometrics  
- MFA  

---

### Authorization

Authorization determines what you are allowed to do.

#### Simple way to remember it
> Authorization = what you can access

#### Examples:
- File permissions  
- User roles  
- Access levels  

---

### Accounting

Accounting tracks what you do.

#### Simple way to remember it
> Accounting = logging and monitoring

#### Examples:
- Logs  
- Audit trails  
- User activity tracking  

---

## 3️⃣ Non-Repudiation

Non-repudiation ensures someone cannot deny an action they performed.

### Simple way to remember it
> Non-repudiation = you can’t deny it

### Examples:
- Digital signatures  
- Email signing  
- Transaction logs  

---

## 4️⃣ Least Privilege

Users should only have the minimum access needed to do their job.

### Simple way to remember it
> Give only the access needed—nothing more

---

## 5️⃣ Need to Know

Access is granted only if required for a specific task.

### Simple way to remember it
> You only get access if you need it

---

## 6️⃣ Zero Trust (High Yield 🔥)

Never trust any user or system by default—even inside the network.

### Simple way to remember it
> Trust nothing, verify everything

---

## 7️⃣ Defense in Depth

Use multiple layers of security instead of relying on one control.

### Simple way to remember it
> Multiple layers = stronger security

---

## 8️⃣ Gap Analysis (Sometimes included)

Identifies the difference between current security and desired security.
