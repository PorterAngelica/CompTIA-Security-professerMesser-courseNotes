# Security+ SY0-701 Study Guide

## Domain 1.1 – Security Controls

---

## 1. What Are Security Controls

A **security control** is something implemented to protect systems, networks, data, or facilities from threats.

Security controls help an organization:

- Prevent attacks  
- Detect attacks  
- Respond to incidents  
- Reduce risk  

### Simple way to remember it
> Security controls are protections put in place to reduce security risk.

Almost everything used in cybersecurity can be considered a security control.

### Examples:
- Firewalls  
- Security policies  
- Access badges  
- Monitoring systems  
- Security guards  

---

## 2. Security Control Categories

Security controls are often grouped into three categories:

---

### Technical Controls

Technical controls are security protections implemented using technology or systems.

These controls are enforced automatically by hardware or software.

#### Simple way to remember it
> Technical controls are security controls implemented by technology.

#### Examples:
- Firewalls  
- Intrusion Detection Systems (IDS)  
- Antivirus software  
- Encryption  
- Multifactor Authentication (MFA)  
- Access control systems  

#### Example:
A company installs a firewall that blocks unauthorized network traffic.  
➡️ That firewall is a **technical control**.

---

### Managerial Controls

Managerial controls focus on planning, policies, and managing security.

They define how an organization approaches security.

#### Simple way to remember it
> Managerial controls define the rules of security.

#### Examples:
- Security policies  
- Risk assessments  
- Security governance  
- Compliance requirements  
- Security planning  

#### Example:
A company creates a password policy requiring complex passwords.  
➡️ This policy is a **managerial control**.

---

### Operational Controls

Operational controls rely on people and processes.

They describe how security procedures are carried out in daily operations.

#### Simple way to remember it
> Operational controls are security performed by people following procedures.

#### Examples:
- Incident response procedures  
- Security awareness training  
- Backup procedures  
- Change management processes  
- Security guards performing patrols  

#### Example:
Employees are trained to recognize phishing emails.  
➡️ This training is an **operational control**.

---

## 3. Security Control Types

Security controls can also be classified based on what they do.

---

### Preventive Controls

Preventive controls stop incidents before they happen.

#### Simple way to remember it
> Preventive controls stop attacks before they start.

#### Examples:
- Firewalls  
- MFA  
- Access control lists  
- Door locks  
- Network segmentation  

#### Example:
A firewall blocks unauthorized traffic entering a network.  
➡️ This is a **preventive control**.

---

### Detective Controls

Detective controls identify when an attack has occurred.

#### Simple way to remember it
> Detective controls detect attacks.

#### Examples:
- Intrusion Detection Systems (IDS)  
- Log monitoring  
- SIEM alerts  
- Security cameras  
- File integrity monitoring  

#### Example:
A SIEM alerts administrators about unusual login activity.  
➡️ This is a **detective control**.

---

### Corrective Controls

Corrective controls fix problems after an incident.

#### Simple way to remember it
> Corrective controls repair damage after an attack.

#### Examples:
- Restoring backups  
- Removing malware  
- Reinstalling systems  
- Applying security patches  

#### Example:
A company restores files from backup after ransomware encryption.  
➡️ This is a **corrective control**.

---

### Deterrent Controls

Deterrent controls discourage attackers.

#### Simple way to remember it
> Deterrent controls try to scare attackers away.

#### Examples:
- Warning signs  
- Visible security cameras  
- Security guards  
- Account lockout policies  

#### Example:
A building displays “This facility is monitored by security cameras.”  
➡️ This is a **deterrent control**.

---

### Compensating Controls

Compensating controls are alternatives when primary controls cannot be used.

#### Simple way to remember it
> Compensating controls replace a security control that cannot be used.

#### Example:
If an application cannot support MFA, the company might:

- Restrict access by IP  
- Increase monitoring  
- Enforce stricter logging  

➡️ These are **compensating controls**.

---

### Physical Controls

Physical controls limit physical access to systems or locations.

#### Simple way to remember it
> Physical controls protect things you can physically touch.

#### Examples include protecting:
- Server rooms  
- Data centers  
- Network equipment  
- Office buildings  

#### Examples of Physical Controls:
- Locks  
- Badge access systems  
- Fences  
- Security guards  
- Security cameras  
- Mantraps  
- Lighting  

#### Example:
A company installs a badge reader on the server room door.  
Only authorized employees can enter.  
➡️ This is a **physical control**.
