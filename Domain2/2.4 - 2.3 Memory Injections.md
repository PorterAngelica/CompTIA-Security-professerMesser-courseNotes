# 🔐 2.3 : Memory injections

---

## 1️⃣ Core Concept: Execution in Memory

### 🧾 Definition:

All software must be **loaded into memory (RAM)** before it can run on a computer

---

### 🧠 Process Flow:

1. Program stored on disk
2. Loaded into memory (RAM)
3. Executed by CPU

---

### ⚠️ Key Idea:

👉 Nothing executes unless it is in memory
👉 Therefore, malware must enter memory to function

---

---

# 2️⃣ What Exists in Memory

---

## 🧩 Common Memory Components

---

### 🔹 Process

A running program (e.g., Chrome, Word)

---

### 🔹 Thread

A smaller unit of execution inside a process

---

### 🔹 Buffer

Temporary memory storage

---

### 🔹 DLL (Dynamic-Link Library)

#### 🧾 Definition:

A **DLL (Dynamic-Link Library)** is a file containing code that multiple programs can use

---

### ⚠️ Key Idea:

👉 These are normal system components
👉 Malware hides inside them

---

---

# 3️⃣ How Malware Runs

---

## 🧨 Method 1: Standalone Process

* Malware runs as its own program

### ⚠️ Weakness:

* Easier to detect

---

## 🧨 Method 2: Process Injection (VERY IMPORTANT)

---

### 🧾 Definition:

**Process injection** is when malware inserts itself into a legitimate running process

---

### 🎯 Purpose:

* Avoid detection
* Blend with normal activity

---

### ⚠️ Key Idea:

👉 Malware hides inside trusted programs

---

---

# 4️⃣ Why Process Injection is Dangerous

---

## 🔺 Privilege Escalation

### 🧾 Definition:

Gaining higher permissions than originally allowed

---

### 🎯 How it happens:

👉 Malware inherits permissions of the process it injects into

---

### ⚠️ Example:

* Inject into admin process → gain admin rights

---

---

# 5️⃣ How Injection Works

---

## 🧠 Memory Structure

Each process has:

* Start memory address
* End memory address

---

👉 Malware inserts itself **between those addresses**

---

### ⚠️ Result:

* Hidden execution
* Hard to detect

---

---

# 6️⃣ DLL Injection (MOST IMPORTANT)

---

## 🧾 DLL = Dynamic-Link Library

---

## 🧾 Definition:

**DLL injection** is when malware forces a process to load a malicious DLL

---

---

## 🧠 Step-by-Step

1. Attacker places malicious DLL on system

2. Adds path (location) to DLL

3. Target process runs

4. Process loads DLL

5. Malicious DLL enters memory

6. Malware executes

---

---

# 7️⃣ Why DLL Injection Works

---

👉 Programs trust DLLs
👉 System loads them automatically

---

### ⚠️ Key Idea:

👉 Malware abuses normal behavior

---

---

# 8️⃣ Detection Challenges

---

👉 Malware:

* Doesn’t appear as separate process
* Looks legitimate
* Blends into memory

---

---

# 9️⃣ Key Terms (NO CONFUSION)

---

## 🧾 Process Injection

Malware hides inside a running program

---

## 🧾 DLL Injection

Malicious DLL loaded into a process

---

## 🧾 DLL (Dynamic-Link Library)

Shared code file used by programs

---

## 🧾 Privilege Escalation

Gaining higher permissions

---

---
