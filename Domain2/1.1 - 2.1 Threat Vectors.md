# 🔐 2.1: Threat Vectors
---

## 1️⃣ What is a Threat Vector?

### 🧾 Definition:

A **threat vector** (also called an **attack vector**) is the method or path an attacker uses to gain access to a system 

---

### 🎯 Key Idea:

* **Threat Actor = WHO attacks**
* **Threat Vector = HOW they get in**

---

---

# 2️⃣ Messaging-Based Threat Vectors

---

## 📧 Email (Phishing)

### 🧾 Definition:

**Phishing** = fraudulent messages designed to trick users into revealing information or clicking malicious links

---

### 🎯 How it works:

* Fake login page
* Malicious link
* Fake invoice

---

---

## 📱 SMS (Smishing)

### 🧾 Definition:

**Smishing = SMS Phishing**
(SMS = Short Message Service)

---

### 🎯 How it works:

* Malicious link via text message

---

---

## 📞 Vishing (Voice Phishing)

### 🧾 Definition:

**Vishing = Voice Phishing**

---

### 🎯 How it works:

* Phone call asking for:

  * Passwords
  * Credit card info

---

---

## 💬 Instant Messaging / Direct Messages

* Attackers communicate directly
* Use **social engineering** (manipulating people)

---

### ⚠️ Key Idea:

👉 Messaging works because it exploits **trust**

---

---

# 3️⃣ Image-Based Threat Vectors

---

## 🖼️ SVG (Scalable Vector Graphics)

### 🧾 Definition:

SVG = **Scalable Vector Graphics**
Image format based on **XML (Extensible Markup Language)**

---

### ⚠️ Important:

👉 SVG is NOT just an image
👉 It can contain code like:

* JavaScript
* HTML

---

## 🧠 XSS (Cross-Site Scripting)

### 🧾 Definition:

**XSS = Cross-Site Scripting**
An attack where malicious scripts run in a user’s browser

---

### 🎯 How it works:

* Code injected into:

  * Webpage
  * Image (like SVG)
* Browser executes it

---

### ⚠️ Key Risk:

* Steal cookies
* Hijack sessions

---

---

# 4️⃣ File-Based Threat Vectors

---

## 📄 PDF (Portable Document Format)

* Can contain:

  * Scripts
  * Embedded objects

---

## 📦 Compressed Files

* ZIP (Zipped file)
* RAR (Roshal Archive)

---

### ⚠️ Risk:

👉 Malware hidden inside

---

---

## 📊 Office Files (Macro Attacks)

### 🧾 Definition:

A **macro attack** is malicious code embedded in a document

---

### 🎯 How it works:

* User opens file
* Macro executes
* Malware runs

---

---

## 🧩 Browser Extensions

* Add-ons/plugins
* Can include malicious code

---

---

# 5️⃣ Voice & Telephony Threat Vectors

---

## 📡 SPIT (Spam over IP Telephony)

### 🧾 Definition:

Spam calls over VoIP systems

---

---

## 🔢 War Dialing

### 🧾 Definition:

Scanning phone numbers to find vulnerable systems

---

---

# 6️⃣ Removable Media Threat Vectors

---

## 🔌 USB Attacks

* Malware stored on USB
* User plugs it in → infection

---

## ⌨️ USB Keyboard Injection

### 🧾 Definition:

USB device acts like a keyboard

---

### 🎯 What it does:

* Automatically types commands
* Executes malware

---

---

## 🌐 Air-Gapped Network

### 🧾 Definition:

A system with **no connection to the internet or external networks**

---

### ⚠️ Important:

👉 Still vulnerable through:

* USB
* Physical access

---

---

# 7️⃣ Software & System Threat Vectors

---

## 🐛 Unpatched Software

* Missing updates
* Known vulnerabilities

---

## 🧨 Zero-Day Vulnerability

### 🧾 Definition:

A vulnerability unknown to the vendor (no patch available yet)

---

---

## 🧓 Unsupported Systems

* No longer receive updates
* High risk

---

---

## 🌐 Web Applications

* Centralized systems
* One compromise → affects all users

---

---

# 8️⃣ Network-Based Threat Vectors

---

## 📡 Wireless Attacks

* Weak protocols:

  * WEP (Wired Equivalent Privacy)
  * WPA (Wi-Fi Protected Access)
  * WPA2
  * WPA3 (latest, most secure)

---

## 📶 Rogue Access Point

### 🧾 Definition:

Unauthorized wireless access point

---

---

## 🔐 802.1X

### 🧾 Definition:

Network authentication protocol

---

### 🎯 Purpose:

* Prevent unauthorized access

---

---

## 🔵 Bluetooth Attacks

* Weak implementations
* Used for reconnaissance

---

---

# 9️⃣ Open Ports & Misconfiguration

---

## 🚪 Open Ports

* Example:

  * Port 80 = HTTP
  * Port 443 = HTTPS

---

### ⚠️ Risk:

👉 Entry points for attackers

---

---

## ⚠️ Misconfiguration

### 🧾 Definition:

Incorrect setup of systems

---

### 🎯 Risk:

* Creates vulnerabilities

---

---

---

## 🔑 Default Credentials

Pre-set login credentials

---

### 📌 Example:

* admin / admin

---

### ⚠️ Risk:

👉 Full system access

---

## 🧑‍💻 MSP (Managed Service Provider)

Third-party company that manages IT systems

# 🧾 ANSWERS – Threat Vectors

---

## 🔹 Flashcards

1. A method used by an attacker to gain access to a system.

2. Threat actor = **who attacks**
   Threat vector = **how they attack**

3. Because they allow **direct communication and social engineering**

4. **Smishing** = phishing via SMS/text messages

5. **Vishing** = phishing via voice/phone calls

6. Because they can contain **embedded code (JavaScript/HTML)** that runs in the browser

7. Because they can contain **scripts and embedded malicious content**

8. A malicious script embedded in an Office document

9. Because they can **hide malicious files inside compressed content**

10. Scanning phone numbers to find systems to exploit

11. Because they can contain malware or act as input devices

12. A network with **no direct internet connection**

13. A vulnerability unknown to the vendor and not yet patched

14. Because they **no longer receive security updates/patches**

15. An unauthorized wireless access point

16. Because they expose entry points into a system

17. Pre-set usernames/passwords (e.g., admin/admin)

18. An attack through a third-party vendor or provider

19. Managed Service Provider (third-party managing systems)

20. Because incorrect configuration can **create security vulnerabilities**
