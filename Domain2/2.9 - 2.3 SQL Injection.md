# 🔐 2.3: SQL Injection (SQLi)

---

## 1️⃣ What is Code Injection?

---

### 🧾 Definition:

A **code injection attack** is when an attacker inserts malicious code into application input so the application executes it 

---

### 🎯 Core Idea:

👉 Application expects **data**
👉 Attacker sends **code instead**

---

---

# 🧠 🔥 ANALOGY (IMPORTANT)

---

## 📝 Form Input Analogy (Correct Way to Think)

Imagine:

* A form asks for your **name**
* Expected input:
  👉 “John”

---

### 🧨 Attack:

You enter:

👉 `John; delete all records`

---

### 🎯 What happens:

* System doesn’t separate:

  * Data ❌
  * Commands ❌

👉 It runs BOTH

---

### 🧠 REAL MEANING:

* Input field = entry point
* Application = blindly trusting input
* Injected code = attacker commands

---

👉 The system cannot tell:

> “This is data” vs “This is code”

---

---

# 2️⃣ Types of Code Injection

---

👉 Common types:

* HTML Injection
* SQL Injection (SQLi)
* XML Injection

---

---

# 3️⃣ What is SQL Injection (SQLi)?

---

### 🧾 Definition:

**SQL Injection (SQLi)** is when an attacker injects SQL code into a database query to manipulate or access data 

---

### 🧾 SQL = Structured Query Language

👉 Language used to communicate with databases

---

---

# 4️⃣ How Applications Normally Work

---

## 🧠 Normal Query

User inputs:
👉 “Professor”

---

Query sent:

```sql
SELECT * FROM users WHERE name = 'Professor'
```

---

👉 Database returns:

* Only matching records

---

---

# 5️⃣ How SQL Injection Works

---

## 🧨 Attack Input

Instead of normal input:

👉 `Professor OR 1=1`

---

---

## 🧠 New Query

```sql
SELECT * FROM users WHERE name = 'Professor' OR 1=1
```

---

---

## ⚠️ Key Insight:

👉 `1 = 1` is ALWAYS TRUE

---

👉 So database returns:

> EVERYTHING

---

---

# 🔥 WHY THIS WORKS

👉 Application does NOT validate input

👉 It directly inserts user input into query

---

---

# 6️⃣ Why SQL Injection is Dangerous

---

👉 Attacker can:

* View all data
* Modify data
* Delete data
* Shut down database

---

### ⚠️ Key Idea:

👉 Attacker gains **full control of database**

---

---

# 7️⃣ Why It’s Easy to Exploit

---

👉 No special tools needed

👉 Just:

* Type malicious input
* Use browser

---

---

# 8️⃣ Real Example (From Video)

---

## 🧠 Normal Input:

* Name = Smith
* Password = 3SL99A

---

👉 Returns only Smith’s data

---

---

## 🧨 Injected Input:

```sql
' OR '1'='1
```

---

👉 Query becomes TRUE for ALL records

---

## 💥 Result:

👉 Entire database exposed

---

Language for databases
