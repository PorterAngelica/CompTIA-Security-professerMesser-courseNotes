# 🔐 2.3: Buffer Overflow Attacks

---

## 1️⃣ What is a Buffer Overflow?

---

### 🧾 Definition:

A **buffer overflow** is an attack where an attacker writes **more data than expected** into a memory space, causing it to overflow into adjacent memory

---

### 🎯 Core Idea:

👉 Memory has limits
👉 Attacker writes beyond those limits

---

---

# 🧠 🔥 ANALOGY (IMPORTANT – THIS IS HOW YOU SHOULD THINK)

---

### 🎒 Suitcase Analogy

* **Buffer = suitcase**
* **Expected data = 8 items**
* **Attacker input = 13 items**

---

### 🧠 What happens:

You force all 13 items into a suitcase meant for 8:

* The suitcase overflows
* Extra items spill into **another suitcase (adjacent memory)**

---

### ⚠️ Now the IMPORTANT part:

👉 The attacker **intentionally chooses what spills over**

So instead of random items, they place:

* **Malicious objects (malicious code)**

---

### 🎯 Final effect:

When the “friend” (the system/program) opens their suitcase:

👉 They unknowingly use the attacker’s malicious items

---

### 🔥 What this REALLY means in computers:

* Your program uses memory next to the buffer
* Overflow changes that memory
* Program behavior is now **controlled by attacker**

---

---

# 2️⃣ What is a Buffer?

---

### 🧾 Definition:

A **buffer** is a temporary storage area in memory used to hold data

---

### 🎯 Example:

* Program expects **8 bytes**
* Buffer size = 8 bytes

---

---

# 3️⃣ What is Bounds Checking?

---

### 🧾 Definition:

**Bounds checking** ensures that data written into memory does not exceed allowed size

---

### ⚠️ Key Idea:

👉 This is what should STOP the suitcase from overflowing

---

---

# 4️⃣ How a Buffer Overflow Happens

---

## 🧠 Step-by-Step

1. Application expects fixed input
2. Attacker sends too much data
3. Extra data spills into nearby memory
4. Memory is overwritten

---

---

# 5️⃣ Why This is Dangerous

---

👉 Overflow can:

* Crash the system
* Modify program behavior
* Inject malicious code

---

### 🔥 Using the analogy:

👉 You’re not just overpacking
👉 You’re **putting dangerous items into someone else’s suitcase**

---

---

# 6️⃣ Attacker Goal

---

👉 Not random overflow

👉 They want:

* Controlled overflow
* Specific memory overwrite
* Predictable results

---

---

# 7️⃣ Real Example (From Video)

---

## 🧠 Memory Layout

* Variable A → 8 bytes
* Variable B → 2 bytes

---

## 🎯 Important:

👉 Variable B controls permissions

* < 2000 → user
* > 24000 → admin

---

---

## 🧨 Attack

Input: **“excessive” (9 bytes)**

---

### 🧠 What happens:

* First 8 bytes → fill Variable A
* 9th byte → spills into Variable B

---

### ⚠️ Result:

👉 Variable B changes value
👉 Now gives **admin access**

---

---

# 8️⃣ Key Concept: Privilege Escalation

---

### 🧾 Definition:

Gaining higher permissions than allowed

---

### 🎯 In this attack:

👉 Overflow modifies memory → increases privileges

---

---

# 9️⃣ Why This Attack is Difficult

---

👉 Not every overflow works

---

### Possible results:

* Crash
* No effect
* Successful exploit

---

---

### 🎯 Attacker needs:

* Precision
* Correct location
* Repeatability
